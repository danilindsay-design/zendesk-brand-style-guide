Brand expressions

You know if it's on brand if it's...

Approachable

Great technology should feel intuitive. That's why we present our brand in a way that's welcoming, clear, and easy to understand—no jargon, no barriers, just solutions designed for everyone.

Modern

Innovation is in our DNA. Through bold, forward-thinking expressions of our brand, we communicate that we're not just keeping up with the future—we're building it.

Trusted

We're advocates for consistency and quality. Every touchpoint reinforces reliability, from the precision of our design to the confidence we foster in the services we provide.

Connected

To lead the way, we stay relevant. Our brand speaks to the needs of our customers and reflects the culture of the moment, ensuring we're not just part of the conversation but shape it.

Fresh

Our visual and verbal identity mirrors the elegance of our solutions. With clean, intuitive design, we show that embedding AI into customer and employee experiences can be as seamless as it is transformative.

The language of resolution

Our design system guides customers from challenge to resolution through clean, intentional choices. Simple forms create clarity, while gradients and shapes signal the path forward. Every visual element reinforces our core purpose: transforming support moments into exceptional experiences.

Layout & Grid System

The Grid

Greenhouse uses a 12-column grid with responsive column gutters, margins, and padding specific to each breakpoint. All elements should align to the grid for consistent layouts and readability.

Breakpoints

Breakpoint Viewport Range Grid Columns Above Fold Target
XS 320px - 480px 12 columns (stacked) 640px
S 481px - 767px 12 columns (stacked) 640px
M 768px - 1119px 12 columns 900px
L 1120px - 1599px 12 columns 880px
XL 1600px+ 12 columns 1080px

Note: The "S (481-767)" viewport range gets minimal traffic and can be omitted in handoff. The "S" breakpoint follows "XS" in terms of grid, while font-size and spacing change.

Responsive Breakpoints

Greenhouse uses a mobile-first responsive approach with five standard breakpoints. Design for the smallest screen first, then progressively enhance for larger viewports.

Responsive Design Principles

Mobile First

Design for mobile devices first, then progressively enhance for larger screens. This ensures core content and functionality work on all devices.

Fluid Typography

Typography scales across breakpoints. Headlines reduce in size on mobile while maintaining hierarchy and readability.

Flexible Grids

Use percentage-based widths and the 12-column grid. Columns stack on mobile and expand into multi-column layouts on larger screens.

Touch Targets

Minimum touch target size is 44x44px for mobile. Ensure adequate spacing between interactive elements on touch devices.

Performance

Optimize images and assets for mobile. Use responsive images with srcset and consider lazy loading for below-the-fold content.

Content Priority

Prioritize content hierarchy on mobile. Essential information and primary CTAs should be visible without scrolling.

Spacing Variables

Greenhouse spacing is organized in two collections: Breakpoint (responsive spacing that adapts to each breakpoint) and Base (fixed spacing for granular control).

Breakpoint Spacing Variables

Always use variables from the Breakpoint collection when possible. These automatically adjust to breakpoints.

text-to-content Used in spacing between text and other elements
content-to-content Used between rows of content within sections
button-margin Used between buttons/links and other elements
section-padding Used for section padding (vertical and horizontal)
container-padding Padding used in containers
tablet-spacer Used to create additional space in the Tablet breakpoint
column-gutter Column spacing in grid layouts

Base Spacing Variables

Use the Base collection for custom spacing when breakpoint variables don't suffice. These values do not dynamically change.

Available in 8px increments: spacer-8, spacer-12, spacer-16, spacer-24, spacer-40, spacer-56, spacer-72, spacer-96, spacer-104, spacer-120, spacer-144, spacer-160

Invisible Spacer Component

In most cases, use Auto Layout to space elements. When that's not possible (i.e., for gaps between columns or stacked content), use the Spacer component (80px default).

Layout Best Practices

✓ Do Balance text with imagery and negative space

Avoid filling every available space with text. Use imagery and white space to create breathing room and improve readability.

✓ Do Consider layout orientation based on content amount

When there's not much content, consider laying it out side by side rather than vertically to avoid too much negative space on one side.

✓ Do Use breakpoint spacing variables for consistency

When possible, use spacing variables from the Breakpoint collection to maintain consistency and ensure spacing adapts responsively to each breakpoint.

✕ Don't Crowd elements too close together

Maintain adequate spacing between elements to improve scannability and visual hierarchy. Use the spacing system consistently.

✕ Don't Use incorrect image sizing relative to content

Ensure images are proportional to the height and amount of content. Oversized or undersized images disrupt visual balance.

✕ Don't Put promos on separate backgrounds

Integrate promotional content into the same background system as the rest of the page for visual cohesion.
