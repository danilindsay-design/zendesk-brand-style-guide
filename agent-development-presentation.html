What is an Agent?
(The Historical Context)

  • The Etymology of "Agency": In 2020 (pre-gen AI), an "agent" was a representative (like a real estate or travel agent) authorized to act on behalf of a principal.
  • The "New Hire / Intern" Mental Model:
    • Start by thinking about the agent as a human and write a Job Description (System Prompt) and a Scope of Authority (Tools).
    • They are capable, but lack your specific "company knowledge" and other nuances of how to perform and evaluate work.
  • Core Concept: AI as a delegated representative with accountability and methods for measuring success or reporting outcomes for external measurement of success
  • You are the principal!

Does psychology apply to AI?

Priming

Priming in psychology is the unconscious, implicit memory process where exposure to a stimulus (word, image, scent) influences responses to later stimuli.

How to use: Mention a person, technology, approach or to activate related parts of the LLM "brain".

Example: In the style of Zig Ziglar, give me a brief motivational email to my sales team.

The IKEA effect

The IKEA effect is a cognitive bias where people disproportionately value, love, or overpay for products they partially created or assembled themselves.

How this factors in: When we use AI to build tools, there is an increased tendency to value our tool/application disproportionately higher than other tools.

Linear vs. Non-linear

  • By default, humans tend to think in linear ways
    • E.g. Do X and Y to product outcome Z
  • Many AI projects are focused on existing linear processes and simply try to replace the human with AI without rationalizing the process
  • Non-linear starts with the business outcome and focuses on the strengths of AI to reimagine processes
  • Humans don't scale well, but AI can

Example: Review Closed Cases

Existing workflow: Supervisors views report of all cases with a 1 or 2 star rating and escalates where needed.

Linear

  • Have AI read the report and evaluate the cases for escalation
  • Generate new report for escalations

Non-linear

  • Let AI review every case, regardless of user rating
  • Assess for alignment of rating and user sentiment and duration or other metrics
  • For each case where there is misalignment or need for escalation, an agent can reopen and assign the case

Triggers & Autonomy

  • Agents are often deployed as any other application: API (REST), but sometimes use a specific protocol.
  • Agents can be executed by: CRON, infinite loop, polling, subscribing, direct API call, CLI, application initiated, etc.
    • Going back to the intern analogy, you might: email, chat, call, text, stop by, etc.
    • An agent may use the above methods to trigger another sub-agent
  • "Autonomous" agents will STILL be called in one of the above ways

Build your first agent... continued

import os
from google.adk.agents.llm_agent import Agent
from google.adk.models.lite_llm import LiteLlm

# 1. Configure the model to use the Zendesk Gateway
model = LiteLlm(
    model='openai/gpt-4o',
    api_base=os.getenv("OPENAI_API_BASE"),
    api_key=os.getenv("OPENAI_API_KEY")
)

# 2. Mock tool implementation
def get_current_time(city: str) -> dict:
    """Returns the current time in a specified city."""
    return {"status": "success", "city": city, "time": "12:30 AM"}

# 3. Initialize the Agent
root_agent = Agent(
    model=model,
    name='root_agent',
    description="Tells the current time in a specified city.",
    instructions="You are a helpful assistant.",
    tools=[get_current_time],
)
